# 賽特選房系統 - 產品需求文件 (PRD)

## 1. 專案概述

### 背景與動機
在各種選房或是娛樂/投資平台中，參與者最大的痛點是無法確保穩定獲利，甚至經常面臨虧本的風險。為了解決這個問題，本專案開發「賽特選房系統」，主打「一定要出金，不能虧本」的終極目標。系統將透過演算法與自動化機制，大幅降低人為操作風險，確保資金的安全性與穩定增長。

### 目標用戶
- 追求穩定獲益，希望避免在選房或投資過程中虧本的玩家或投資客。
- 沒時間全天候盯盤，需要依賴自動化工具代為操作的用戶。
- 依賴數據分析，希望能提早掌握「爆分（大獎）」時機的進階使用者。

### 核心價值主張
零虧損風險、全自動化操作、以及即時的爆分預警系統，幫助用戶實現穩定的資金增長並安全出金。

---

## 2. 功能需求

以下列出系統的核心功能，並以使用者故事（User Story）格式描述：

1. **智能選房**
   - **作為** 玩家，**我希望** 系統能結合相關數據與演算法自動評估並選出勝率最高、最有利的房間，**以便** 大幅提高我的獲利機率。

2. **穩定出金（出金保固機制）**
   - **作為** 玩家，**我希望** 系統具備嚴格的資金水位監控與穩定的自動出金機制，**以便** 當獲利達到設定目標時能迅速落袋為安，確保絕對不虧本。

3. **自動設定停損點**
   - **作為** 玩家，**我希望** 系統可以根據我的總資金或當前狀況，自動精算並設定嚴格的停損點，**以便** 於情況不如預期時能瞬間切斷損失，防禦本金。

4. **自動操作 (掛機模式)**
   - **作為** 玩家，**我希望** 系統可以代為自動排滿及操作所選定的房間，**以便** 我不需要 24 小時守在螢幕前也能持續運作並獲利。

5. **快爆分時提前通知**
   - **作為** 玩家，**我希望** 系統能在監測到某房間即將達到「爆分（大滿貫預兆）」前發布推播通知或自動加大操作，**以便** 讓我能抓準時機，最大化整體收益。

---

## 3. 非功能需求

### 技術限制
- **後端框架**：使用 Python 的 Flask 框架開發。
- **前端模板**：使用 Jinja2 渲染動態視圖，配合 HTML/CSS/JS。
- **資料庫**：使用 SQLite 輕量級資料庫進行資料存儲與管理。

### 效能與安全考量
- **效能**：系統需具備高強度的即時監控能力，特別是在「快爆分通知」與「停損點自動觸發」上，系統回應時間需控制在毫秒級別。
- **安全性**：牽涉到「出金」與「自動操作」等核心資金相關功能，系統必須保護用戶設定檔與系統憑證的安全性，並防範惡意的注入攻擊或未授權訪問。
